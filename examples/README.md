# Examples

These are real artifacts from this project's own development, lightly sanitized. They show *what your own folders will look like after using the system for a while.*

- `worklogs/` — example daily worklog showing the goal/tasks/work-log/summary structure.
- `planning/items/` — example planning backlog items at different statuses.
- `state/items/` — example state items (questions, hypotheses, findings, assumptions, decisions).
- `planning/INDEX.md`, `planning/horizons.md` — pre-rendered dashboards generated by `make planning-index` against these example items.
- `state/INDEX.md`, `state/{open_questions,hypotheses,findings,assumptions,decisions}.md` — pre-rendered dashboards generated by `make state-index` against these example items.

**You don't edit these.** They're frozen reference snapshots. Your own data lives at the paths you configured during `/setup`, not here.

If you want to see how the dashboards regenerate against your own items, run `make planning-index` (or `make state-index`) from your `<notes-dir>` after creating an item there.
