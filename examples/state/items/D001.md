## Claim / Question / Decision

Skills are co-located with the project under `.claude/skills/` rather than stored in a shared global location. The repo ships with its own skill set and config.

**Rationale:** A shared global skill location would create external dependencies and make cross-machine portability fragile. Project-level skills can be copied to any new project as-is; the entire skill system travels with the repo.

**Consequences:**
- Skills are version-controlled alongside the project's other docs
- Teams using the same repo get identical skill behavior
- No global install or external dependency is required
- Setup is: copy `.claude/skills/` + run `/setup`
