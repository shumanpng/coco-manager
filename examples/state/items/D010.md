## Claim / Question / Decision

`/end-week` is purely retrospective: it summarizes what happened and fills `## Review notes`, but makes no carry-forward suggestions. `/build-week` is purely prospective: it builds the next plan but does not look back. The two skills do not overlap.

**Rationale:** Prevents the skills from stepping on each other's output. Carry-forward is explicitly `/build-week`'s responsibility; retrospective is `/end-week`'s. Each skill does one thing well. The user runs them in sequence at week boundaries.

**Consequence:** `/end-week` has no "next week" section. If a user wants to carry items forward, they run `/build-week` afterward.

## What would change my mind

If users consistently find themselves wanting carry-forward suggestions inside `/end-week`, the boundary could be softened (e.g., a "suggested items for next week" section that is explicitly non-binding).
