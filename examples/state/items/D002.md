## Claim / Question / Decision

A single `meta.md` file at `.claude/skills/meta.md` holds all path configuration for the skill system. `/setup` is the only skill with write access to it; all other skills are read-only.

**Rationale:** One file to change per project means moving to a new project is a single edit (or a `/setup` run). Skills that detect `<not set>` guide the user to `/setup` rather than silently failing. Read-only skills minimize the surface area for accidental writes.

**Consequences:**
- First-time setup requires one `/setup` run — invisible on every subsequent invocation
- `meta.md` is the single file to inspect when debugging path issues
- Skills cannot accidentally modify config (no write tools except in `/setup`)

## What would change my mind

If path configuration becomes complex enough (e.g., per-user overrides, environment-specific paths) that a single flat file is insufficient — then a more structured config format or environment variable approach would be warranted.

No action needed — this decision is implemented and stable.
