## Claim / Question / Decision

Constraints are written into the `## Constraints` section of the weekly plan file (`planning/weekly/YYYY-Www.md`), not a separate `constraints.md`. Standing constraints carry forward from the prior week's plan. If no prior plan exists, `/build-week` asks the user whether to specify constraints now or defer.

**Rationale:** Reduces file proliferation; keeps constraints co-located with the plan they govern; simplifies `/build-week` (reads one file instead of two). A separate `constraints.md` duplicated information already expressible in the weekly plan.

**Consequence:** `planning/constraints.md` was deleted. The weekly plan template (`planning/_weekly_plan_template.md`) was updated to include a dedicated `## Constraints` section.

## What would change my mind

If constraints become complex enough that they need cross-week tracking independent of the plan (e.g., multi-week hard deadlines referenced by multiple plans), a separate file could be justified.
