## Claim / Question / Decision

A weekly plan is considered closed/complete when its `## Review notes` section has substantive content. No YAML frontmatter status field is used to track open/closed state.

**Rationale:** Weekly plan files don't currently use YAML frontmatter; adding it would be a structural change for minimal benefit. The presence of review notes is self-documenting and easily scanned. `/end-week` uses this as its idempotency check: if `## Review notes` already has content, it warns before overwriting.

**Consequence:** `/end-week` must read the `## Review notes` section body (not a frontmatter field) to determine whether a plan is already closed.

## What would change my mind

If weekly plans adopt YAML frontmatter for other reasons (e.g., generated index tooling that needs machine-readable status), migrating to a `status: closed` field at that point would be natural.
