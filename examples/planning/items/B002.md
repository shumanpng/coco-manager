### Problem / Goal

New (and existing) users have no conversational way to discover available skills and their purpose. Questions like "what skills are available?" require reading through skill files manually.

### Proposed approach

Create an `/about` skill that reads no external file — it embeds a static skill summary table directly in the SKILL.md body, covering every shipped skill plus `/about` itself.
