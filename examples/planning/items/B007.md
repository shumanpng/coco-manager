### Problem / Goal

No skill exists for weekly wrap-up. Users have no structured way to summarize the week's work, reflect on progress against the weekly plan, and formally close the `YYYY-WWW.md` file — leaving weekly plans dangling without a resolution step.

### Proposed approach

Single-phase skill: read weekly plan + worklogs + INDEX.md + meeting notes → draft `## Review notes` section → confirm with user → write. Includes a lightweight planning item reconciliation pass (safety net for missed `/end-session` status updates). Idempotent: detects existing review notes and asks before overwriting. Modeled on `/end-session` and `/build-week` skill patterns.

### Dependencies / blockers

None.

### Next smallest step

SKILL.md drafted at `.claude/skills/end-week/SKILL.md`. Next: test by running `/end-week` against a project with an existing weekly plan.

### Success looks like

Running `/end-week` fills `## Review notes` in the current week's plan file with a structured review (priorities outcome, planned work recap, unplanned work, risks, learnings). Re-running is safe (detects existing content and asks before overwriting). Lightweight planning item reconciliation catches missed status updates.
