### Problem / Goal

No skill exists for creating or updating this week's plan. Users have to manually create and edit the weekly plan file, which is the same friction that motivated the original slash-command system.

### Proposed approach

Create a `/build-week` skill (also acceptable as `/build-weekly-plan`; `/build-week` is currently preferred) that reads the planning and state indices to suggest prioritized items for the week, writes (or updates) the current `YYYY-W##.md` file in `weekly-planning/`, and handles the case where constraints change mid-week (either modify in place or create a new version — decide during implementation).

### Dependencies / blockers

None.

### Next smallest step

Test re-run after a constraint change: modify a constraint in the current week's plan, re-run `/build-week context-manager`, inspect the diff output (Step 8b) to verify it reflects the change without losing the original plan.

### Success looks like

Running `/build-week context-manager` (or `/build-weekly-plan`) creates or updates this week's plan file with a prioritized list drawn from the planning index; re-running after a constraint change reflects the update without losing the original plan.
